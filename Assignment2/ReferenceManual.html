Reference Manual, <a> Tag

Abstract

The anchor tag is used to link to both internal and external content. It can explicitly direct to a websites URL, or it can direct to an element via it's ID tag. It can also link directly to files hosted at a URL.

Additionally, it can use protocols other than http. An anchor tag can be used to open a connection over SSH, prompt the client to create an email, open an FTP connection, and more. The result of clicking these links is dependent on how the user has configured their browser to treat such protocols.

Details

href
The URL or ID that the link will access.
Can be of any browser supported protocol.
target
The location where the URL will be opened within the browser environment.
The browser environment consists of browsing contexts, each of which could be a window, tab, or <iframe>
_self: default, loads the URL in the current context(normally window or iframe).
_blank: loads the URL in a new tab or possibly window (browser dependent)
_parent: loads the url and the parent context, e.g. if redirecting the browser from within an iframe
_top: loads the URL to the topmost parent's browsing context
rel
A list of 'link type' strings that defines the relationship between the current page and the linked content.
This is often used with the value noopener noreferer to prevent an external website from hijacking the browser context.
title
The string displayed then the anchor tag is hovered over.
